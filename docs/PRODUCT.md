## 零、品牌定位與核心隱喻：牧羊人的拐杖

Hither 的品牌核心隱喻是**牧羊人的拐杖 (Shepherd's Crook)**：
- **核心意象**：像牧羊人一樣，把走散、迷失的羊群引導並聚集在同一個地方。
- **角色定位**：
  - **Leader (隊長)**：即牧羊人，負責規劃路線、引導方向、決定集合點。
  - **Follower (隊員)**：即羊群，輕鬆跟隨指引移動，不走散、不迷失。
- **視覺與設計指導原則**：
  - 全 App 介面**禁用 Emoji**（唯一的例外是用戶個人頭像的 Emoji 選擇器）。
  - 所有原本使用 Emoji 的元件，必須改用風格統一、極簡且具備質感的**自製線條或填色圖示 (Icons)**。
  - 設計與動畫效果需傳達「安心、指引、凝聚」的意象。

## 一、一句話定位

**Hither 是一群人一起出遊時，讓「不走散、找得到彼此、行程對得上、時間對得齊」的隨身指揮中心。**

願景是明確的：**出國自由行、家庭出遊在規劃與移動的每一刻，都會像打開地圖一樣自然地打開 Hither。** 地圖回答「這個地方在哪」，Hither 回答「我這群人現在在哪、要去哪、什麼時候到、誰需要被引導」。

要達到「不可或缺」，判準只有一個：**它必須在一趟旅程的每一天、每一段都被打開，而不是只有走散那一刻。**

## 二、首次使用體驗 (Onboarding) 與互動分支

為了大副降低用戶心智負擔，同時收集關鍵的旅行背景偏好，Onboarding 流程採用**低文字密度、動畫優先、分支互動式**的設計。

Onboarding 設計的完整 UI 佈局、手勢、動態與震動回饋規範詳見 [onboarding_design_specs.md](file:///C:/Users/alexs/Desktop/BZ/hither/hither_app/docs/../brain/6fd5d31c-11d3-43e7-9e28-54f672ae2d7f/onboarding_design_specs.md)。

### 2.1 共享流程頁面
1. **第一頁：核心價值動畫**
   - 展示牧羊人拐杖將四散的圓點引導聚集到地圖中心的意象。
   - 提供右上角「跳過 (Skip)」按鈕。
2. **第二頁：主題色選擇 (Theme Selector)**
   - 介於核心動畫與角色選擇之間。
   - 用戶可於 `night` (深夜)、`day` (日光)、`dusk` (黃昏) 以及新增的 `forest` (深綠森林) 四種預設主題色卡中選擇，選中時背景顏色立即平滑漸變。
3. **第三頁：角色選擇**
   - 提供「隊長（牧羊人：規劃行程）」與「隊員（羊群：輕鬆跟隨不走散）」兩大視覺卡片。
   - 底部設有低調連結：「我只是先下載看看（先不選擇）」。

### 2.2 互動分支路線

#### 路線 A：隊長流程 (Leader Route)
- **L1（旅行目的）**：提供 2x2 圖示網格，供用戶選擇本次出行目的（如出國旅行、城市探索等）。
- **L2（行程天數）**：使用滑桿 (Slider) 拉動選擇天數 (1 ~ 14天)，每滑動一格觸發輕微觸覺回饋。
- **L3（出發時間）**：精簡版日曆選擇預計出遊時間以顯示倒數天數，或可點選「我現在就要用」。

#### 路線 B：隊員流程 (Member Route)
- **旅行性格測驗 (F1 - F3)**：包含 3 題趣味性的場景式二選一選擇題（如「自由活動時間會衝去排美食還是漫無目的散步」）。
- **性格吉祥物分析 (Mascot Summary)**：測驗完成後，顯示用戶的旅行性格動物吉祥物分析結果，例如：
  - *冒險黃金獵犬 (The Adventurous Golden Retriever)*：高行動力、探索導向。
  - *悠閒無尾熊 (The Chill Koala)*：慵懶放鬆、享受慢步調。
  - *細節邊境牧羊犬 (The Detail-Oriented Border Collie)*：守時、重視規劃。
  - *特立獨行貓 (The Spontaneous Cat)*：獨立探索、定點集合。
  - 結果會附帶「最適合搭配什麼類型的隊長」。
- **F4（景點偏好）**：多選網格（美食、景點、購物、自然等）。
- **分享提示**：性格與偏好資料會跟隨用戶。**當隊員成功輸入 Group Code 加入群組時，App 會自動彈出引導提示，問是否要將這份旅行性格與景點偏好分享給隊長**，以供行程規劃參考。

#### 路線 C：先看看流程 (Browser Route)
- 針對不想扮演特定角色的用戶，提供 3 步極簡互動調查（例如「為什麼想用 Hither」、「通常跟誰出遊」、「最想看什麼功能」），以維持互動感，並於完成後進行背景資料收集。

### 2.3 註冊與匿名引導（漸進式披露）
- 結束 Onboarding 分支後，進入註冊頁面，提供帳號密碼註冊及第三方登入。
- 底部提供低調的「以匿名身份繼續」文字連結。
- 用戶點擊匿名時，才彈出確認 Modal（漸進式披露限制）：
  - **限制條款**：1. 無法保留歷史紀錄；2. 無法跨裝置同步；3. 無法被他人加入常用同伴。
  - **3 天生命週期警告**：明確提示「**匿名帳號與其資料最多保留 3 天**。如果您的出遊行程大於 3 天，建議建立註冊帳號以避免中途遺失群組與定位資料。」

## 三、用戶身分與資料留存邏輯

為了保證極致的流暢體驗與資料安全性，資料狀態轉移遵循以下規則：

1. **重開 App 邏輯**：
   - 用戶完成 Onboarding 後，本地寫入標記，重開 App 直接進入地圖主畫面，**絕不重複顯示 Onboarding**。
   - 所有在 Onboarding 選擇的角色與偏好資料已同步於雲端或本地快取。
2. **匿名轉註冊 (Upgrade)**：
   - 匿名狀態下，角色選擇與測驗偏好已綁定在匿名 Profile 上。
   - 用戶在設定中進行註冊時，資料庫端必須將其 Email/OAuth 帳號直接與該匿名 Profile 進行帳號綁定連結，**不得丟棄或重設 Onboarding 資料，確保無縫升級**。
3. **重設與丟棄**：
   - 設定 (Settings) 頁面提供「**重設旅行偏好**」按鈕，點擊後清除雲端 Profile 的性格測驗與目的天數資料，並允許用戶重新運行此偏好問卷。
4. **匿名 Session 清理**：
   - 後端保留針對 3 天以上未活躍的匿名帳號及其歷史關聯資料進行自動清理的機制。

## 四、地圖視覺、手勢與 3D 動態優化

地圖使用 Apple Maps (MapKit) 作為基礎，並引入進階的 3D 視覺與互動體驗。

### 4.1 獨立地圖色彩模式
- 地圖的亮色 (Light) / 暗色 (Dark) 模式為**獨立設定**，用戶可在設定中自由切換，不受 App 系統主題（如 dusk / night）的強制牽引，提供更強的介面對比度。

### 4.2 地點新增 3D 環繞旋轉動畫
- 當地圖上新增完一個地點（例如隊長手動新增集合點，或隊員建議被隊長批准接受時），地圖應展示滑順的過場動畫：
  1. 將地圖相機自動切換成 **3D 俯視斜角（Pitch 介於 30 至 60 度之間）**，顯示周遭立體建築。
  2. 圍繞該新增的地點進行 **360 度慢速水平旋轉（Heading 變更）** 的動態展示。
  3. 旋轉完畢後平滑復原至普通視角。

### 4.3 雙指傾斜手勢優化
- 優化地圖的雙指垂直平滑手勢判定靈敏度。
- 當兩指在螢幕上垂直滑動（調整傾斜視角）時，設定手勢鎖定閾值，**避免將調整傾斜角的操作誤判為地圖的拖移 (Pan) 動作**，提升手動調整視角的流暢度。

## 五、批次景點匯入規格 (KML 匯入)

為解決從電腦 Google Maps 移轉景點的痛點，App 支援 Google My Maps (我的地圖) 的 KML 檔案匯入：

1. **UI 入口**：
   - 匯入按鈕應放置於地圖搜尋欄旁的「**編輯集合點順序 (行程清單)**」介面中，或者在搜尋欄輸入時提供明顯的「匯入 KML 檔案」選項。
2. **引導操作**：
   - 點擊匯入後，App 內提供簡潔的圖文教學，提示用戶：「在 Google 我的地圖中規劃景點 → 匯出 KML 檔案 → 透過此處上傳」。
3. **免費限制與轉換機制**：
   - 對於**免費帳號，系統僅會批次匯入 KML 檔案內的前 5 個景點**（對應免費版集合點上限），超出部分在列表中以灰色鎖定狀態呈現，並顯示升級 Hither Pro 提示。
   - 匯入時，App 在後端提取 KML 中的景點名稱與座標進行精簡儲存，不保留冗餘的 KML 標記。
4. **未來方向**：
   - 簡化儲存後的座標與名稱資料，未來可支持用戶一鍵將此景點清單儲存至雲端並分享至社群，形成公開的推薦行程。

## 六、自建數據日誌與報錯回饋機制

本產品**不使用第三方分析與監控工具**（如 PostHog, Sentry），所有行為日誌與報錯訊息直接自建於 Supabase 資料庫，以便後續靈活處理。

### 6.1 用戶行為數據記錄 (Activity Logs)
- 系統需自動捕獲並在 Supabase 資料表記錄關鍵行為，包含：
  - 用戶背景屬性（Onboarding 中的角色、目的、性格測驗答案、景點偏好、主題色選取、帳號類型）。
  - 行為事件（Onboarding 的步驟切換、跳過、完成時間；群組建立/加入；集合點設定/建議；Carousel 滑動等）。
  - 自動記錄未處理的異常 (Exceptions) 與系統錯誤訊息。

### 6.2 三角形警示報錯按鈕 `⚠️`
- 取消搖晃回報。在 App 四個關鍵入口設置明顯的 **`⚠️` 三角形警示報錯按鈕**：
  1. **地圖主畫面**（回報定位與地圖繪製問題）。
  2. **行程 Carousel 清單/編輯順序介面**（回報行程編輯與排序問題）。
  3. **KML 匯入介面**（回報解析與檔案上傳問題）。
  4. **設定 (Settings) 頁面**（回報一般性系統與帳號問題）。
- **精準定位**：點擊不同位置的 `⚠️` 時，報錯視窗應自動帶入對應的 `context_tag`。
- **回報內容**：用戶可輸入描述，系統在背景使用畫面擷取模組（如 View Shot）取得當前螢幕截圖，連同裝置型號、系統版本、App 版本一同寫入 Supabase 反饋資料表，並將截圖上傳至資料庫的存儲空間。

## 八、權限與下放系統

預設情況下，行程的管理權限在隊長手中。隊長可在設定中開啟或關閉以下權限：

| 功能 | 預設權限 | 是否允許下放隊員 | 說明 |
|------|:-------:|:--------------:|------|
| 建議集合點 | 隊長專屬 | ✅ 可開啟 | 隊員搜尋並建議景點，隊長收到通知決定接受/拒絕 |
| 直接新增集合點 | 隊長專屬 | ✅ 可開啟 | 隊員新增的地點直接進入行程清單，免審核 |
| 更改集合點資訊 | 隊長專屬 | ✅ 可開啟 | 修改景點名稱或備註描述 |
| 調整集合點順序 | 隊長專屬 | **❌ 不開放** | **只有隊長可以拖動重排集合點順序** |
| 發送 Quick Command | 隊長專屬 | ✅ 可開啟 | 允許隊員發送快速呼叫指令 |

## 十、刻意先不做（守住焦點）

- ❌ **AR 路徑導引**：炫技大於實用，不解痛。
- ❌ **勳章 / 成就 / 遊戲化**：對「不走散、安全、時間約定」的核心價值無貢獻。
- ❌ **社群動態牆**：偏離私密出遊定位。
- ❌ **AI 自動排行程**：先做好 KML 導入與協作。
- ❌ **步行 step-by-step 導航**：交給既有地圖 App（Apple Maps），Hither 不重造導航輪子。
- ❌ **網頁版地圖與管理**：前期專注 iOS/Android 實體移動場景，網頁版暫不考慮。
- ❌ **藍牙離線定位溝通**：跨品牌藍牙中繼通訊技術難度與硬體限制大，暫不考慮。
- ❌ **錯誤警示時的 Haptic 震動**：錯誤或失敗提示不應加載觸感回饋。
